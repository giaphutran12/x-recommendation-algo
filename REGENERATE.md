# Regenerating the Supabase Database

This guide walks you through recreating the entire Supabase database from the codebase. The app requires a live Supabase project — there's no local fallback.

## What's Already in the Codebase

The following data and logic are committed to the repository and don't need to be regenerated:

- **Database schema**: `supabase/migrations/` contains 2 SQL migration files that define all tables, columns, indexes, and RLS policies
- **200 persona definitions**: `src/lib/seed/personas.ts` — user profiles with interests, engagement patterns, and follow preferences
- **~50K pre-generated tweets**: `src/lib/seed/generated/batch-01.json` through `batch-10.json` — tweet text, author info, and metadata
- **Follow graph generator**: `src/lib/seed/follow-graph.ts` — logic to build follower/following relationships based on persona interests
- **Engagement simulator**: `src/lib/seed/engagement-simulator.ts` — generates ~338K engagement records (likes, retweets, replies) based on persona behavior
- **Trained ONNX model**: `models/two_tower.onnx` and `models/feature_config.json` — neural network for engagement prediction

## What's NOT in the Codebase

These are generated fresh during seeding:

- **Gemini embeddings**: 1536-dimensional vectors for each tweet, generated by calling the Gemini API during the seed run
- **Runtime UUIDs**: All user, tweet, and engagement IDs are generated fresh each seed run (doesn't matter for this single-user demo)

## Prerequisites

Before you start, have these ready:

- A Supabase account (free tier works fine)
- Supabase CLI installed: `npm install -g supabase`
- Bun installed (comes with the project, or `curl -fsSL https://bun.sh | bash`)
- (Optional) Google AI API key for embeddings — get one free at https://ai.google.dev/

## Step-by-Step Regeneration

### 1. Create a New Supabase Project

1. Go to https://supabase.com/dashboard
2. Click "New project"
3. Choose a name, region, and password
4. Wait for the project to initialize (2-3 minutes)

### 2. Get Your Project Credentials

1. In the Supabase dashboard, go to Settings → API
2. Copy these three values:
   - Project URL (e.g., `https://xxxxx.supabase.co`)
   - Anon public key
   - Service role key (keep this secret)
3. Also copy the project ref from the URL or Settings → General (e.g., `xxxxx`)

### 3. Link the Project Locally

```bash
supabase link --project-ref <your-project-ref>
```

When prompted, enter your Supabase password.

### 4. Update Environment Variables

Create or update `.env.local` in the project root:

```
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key-here
GOOGLE_AI_API_KEY=your-google-ai-key-here
```

The `GOOGLE_AI_API_KEY` is optional — if you skip embeddings, the app still works but Out-of-Network retrieval falls back to popularity-based ranking.

### 5. Push Database Migrations

```bash
supabase db push
```

This creates all tables, columns, indexes, and RLS policies. You should see output confirming the migrations applied.

### 6. Run the Seed Script

**With embeddings (recommended, ~5-7 minutes):**

```bash
bun run seed
```

This loads 50K tweets, generates 1536-dimensional embeddings via Gemini, creates 200 personas, builds the follow graph, and simulates 338K engagements. Costs ~$0.38 in Gemini API calls.

**Without embeddings (faster, ~2 minutes):**

```bash
bun run seed -- --skip-embeddings
```

The app works fine without embeddings. Out-of-Network source falls back to popularity-based retrieval instead of semantic similarity search.

### 7. Verify the Setup

```bash
bun dev
```

Open http://localhost:3000 in your browser. You should see a feed of tweets. The hardcoded viewer user UUID is `00000000-0000-0000-0000-000000000001`.

## How Embeddings Work

The seed script:

1. Loads tweet text from the local batch JSON files (`src/lib/seed/generated/batch-*.json`)
2. Sends text to Gemini `gemini-embedding-001` API in batches of 100 tweets
3. Requests 1536-dimensional vectors (Matryoshka dimension)
4. Writes vectors to the `tweets.embedding` column in Supabase
5. Runs ~500 API calls total (50K tweets / 100 per batch)

If you skip embeddings with `--skip-embeddings`, the `embedding` column stays NULL. The Out-of-Network source detects this and uses popularity-based retrieval instead of semantic similarity.

## For AI Agents

Run these commands in order:

```bash
# 1. Create project at https://supabase.com/dashboard, get project ref

# 2. Link project
supabase link --project-ref <ref>

# 3. Update .env.local with NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY

# 4. Push migrations
supabase db push

# 5. Run seed (with embeddings)
bun run seed

# 6. Verify
bun dev
# Open http://localhost:3000
```

## Time Estimate

- Project creation: 2-3 minutes
- Linking and env setup: 1 minute
- Migrations: 1 minute
- Seeding with embeddings: 5-7 minutes
- **Total: ~10 minutes**

Without embeddings, seeding takes ~2 minutes, bringing the total to ~7 minutes.
